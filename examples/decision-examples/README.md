# Decision Examples

Illustrative Architecture Decision Records (ADRs) demonstrating how a
reference implementation explains a consequential engineering choice.

## Planned Decisions

| Topic | Example decision scope |
| --- | --- |
| Runtime model | Choose Compose, Swarm or Kubernetes for a documented context |
| Stateful dependency | Managed service versus self-hosted storage/database |
| Edge routing | Reverse proxy and TLS ownership |
| Repository/deployment model | Application repository versus environment configuration boundary |
| Observability depth | Introduce tracing or retain logs/metrics-only baseline |

Examples here must follow the
[Architecture Decision Record Standard](../../architecture/decision-records/README.md)
and clearly state that the decision applies to its example context rather than
serving as a universal mandate.
